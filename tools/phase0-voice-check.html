Phase 0: Voice enumeration / network spike

Reference: docs/SOUND-PLAN.md §6. This is a throwaway diagnostic, not part of the shipped app -- it exists only to answer one question before any real sound-warnings code gets built: does this device have a genuinely offline speech voice, and does merely asking for the voice list ever touch the network?

Before clicking the button below:
  1. Physically disconnect this device from the network -- turn off Wi-Fi (and unplug Ethernet if applicable). Airplane mode is fine and simplest.
  2. Open DevTools (F12 or Ctrl+Shift+I) and switch to the Network tab. Leave it open and visible.
  3. Open this same page two ways and repeat the check for each: directly as a file:// URL (double-click the file), and via app/serve-field.cmd's localhost server.
  4. Repeat the whole thing on at least one Android phone too, in whichever browser you'd actually use in the field (likely Chrome).
#namelanglocalService
What to look for:
  • Did any request appear in the Network tab just from clicking the button, with the network physically off? If your browser is genuinely offline, any request that shows up will fail/hang -- but on some setups a DNS lookup or connection attempt can still be visible in the Network tab even when it can't complete. Note anything you see there, even a failed one.
  • Does the table below show at least one row with localService: true? If every row says false, or the table is empty, that's the "no usable local voice" outcome the plan's decision gate describes.
Report back what you see for each of the four combinations (laptop file://, laptop localhost, phone file://, phone localhost) -- that's everything needed to decide how sound/audioEngine.ts/voices.ts get wired up next.
